# dotfun Website Expansion Project

Purpose: expand dotfun.co into a three-wing architecture while preserving the Grepr/Xurrent workbook quality system.

## Confirmed brand architecture

- **dotfun** is the parent brand.
- **Iris** is software developed by dotfun.
- The website expansion should support three public wings:
  - **Iris Platform** — software with feature pages.
  - **dotfun AI Solutions** — AI implementation/automation solutions with solution pages.
  - **dotfun Growth Services** — growth services with both solution pages and service pages.

## Operating rule

Reuse Grepr process quality, not Grepr messaging. dotfun strategy must be grounded in dotfun-specific brand, positioning, voice, crawl, competitor, and SEO research inputs.

## Key folders

- `reference-docs/` — approved or provided source documents.
- `intake/` — walkthrough and source inventory.
- `brand/` — brand guidelines and voice rules.
- `wings/` — wing-specific positioning.
- `research/` — crawl, competitor, SERP, SEMRush/GSC/GA4 inputs.
- `best-practices/` — Grepr-derived methodology, QA gates, subagent setup.
- `workbooks/` — workbook-first outputs.
- `outputs/` — final handoff artifacts.
